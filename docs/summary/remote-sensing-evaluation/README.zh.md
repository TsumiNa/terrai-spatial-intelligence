## 已经接入：Google Satellite Embedding V1

当前两个示范区已完成 2023 与 2024 年 10 m、64 维嵌入的真实裁剪：

- 横滨：7,820 个有效像素，平均年度余弦变化 0.00969，P95 0.01606。
- 茂原：19,877 个有效像素，平均年度余弦变化 0.02278，P95 0.04355。
- 输出：年度变化热图、2024 相似表征 RGB、100 m 聚合证据单元。

嵌入的 64 个轴不能逐轴解释。页面的相似表征 RGB 来自主成分预览，年度变化为单位向量的 `1 - dot(v2023, v2024)`。两者都不直接进入业务评分，避免把尚未完成本地 hold-out 验证的表征当成地表事实。

## Dynamic World：评估后移除

数据本身为 CC BY 4.0，但 TerrAI 商业使用 Earth Engine 会产生平台计算费用。为满足当前“零额外数据库与分析服务采购”约束，Dynamic World 已从 Demo、生成管线和待接入队列移除。需要可解释静态土地覆盖时评估 ESA WorldCover；需要近期地表/季节变化时使用 Sentinel-2 L2A 本地处理。

## 下一阶段：Sentinel-2 L2A 验证

数据通过 Copernicus Data Space STAC API 检索，优先选择：

- 同一季节的无云或低云影像；
- `SCL` 场景分类层用于云、云影和雪掩膜；
- B02/B03/B04/B08/B11/B12 波段；
- 茂原研究范围外扩 2–5 km，避免边缘效应。

计划产品：

1. **NDVI**：识别活跃植被和持续耕作农地，作为农地转换敏感度证据。
2. **NDWI/MNDWI**：识别水面和季节性湿润区域，补充 OSM 水体退距。
3. **NDBI/裸地指数**：辅助区分已建设、裸地、耕地和潜在棕地。
4. **多时相变化**：检测土地覆盖变化、施工扰动和灾后变化。
5. **质量标志**：记录影像日期、云量、像元分辨率和合成窗口。

Sentinel-2 的 10m 像元适合茂原农地和大场地分析，但不适合直接判断横滨单栋屋顶结构或单条道路损伤。横滨屋顶仍应以高分辨率正射影像、建筑轮廓和现场/无人机资料为主。

官方目录：https://documentation.dataspace.copernicus.eu/APIs/STAC.html

## 场景分工

### 横滨城市韧性

- 高分辨率影像：屋顶形态、周边植被和现场核验底图。
- DEM/起伏/坡度：边坡、道路纵坡、谷地和低点。
- Sentinel-2：只用于区域级绿地、热环境或灾后变化背景，不下放到单栋屋顶评分。

### 茂原新能源开发

- 高分辨率影像：候选地当前利用状况和道路入口。
- Sentinel-2：耕作活跃度、水湿条件、季节变化与裸地识别。
- DEM/起伏/坡度：土方、排水、侵蚀和施工可达性。
- 后续再与宗地、农地、保护区、地价和电网CSV做空间联算。

## 生产化质量规则

- 任何遥感评分必须保留采集日期、传感器、波段、分辨率和云掩膜方法。
- 优先使用季节合成而不是任意单景影像。
- 视觉底图与分析栅格分开管理：好看的影像不自动成为模型证据。
- 生成的指数只作为筛查特征，土地性质和许可状态仍以主管机关数据为准。
