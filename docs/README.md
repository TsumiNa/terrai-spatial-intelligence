`docs/` 只使用以下五类目录；除本索引三语版本外，不在 `docs/` 根部放文档。

| 目录 | 内容 |
|---|---|
| [`architecture/`](architecture/FRONTEND_BACKEND.md) | 当前生效的系统边界、组件职责与调用结构 |
| [`refactor/`](refactor/fl-sl-al-platform/00-overview.md) | 每次重构的背景、决策理由和按 PR 拆分的具体计划 |
| [`data/`](data/README.md) | FL 已接入数据集逐项来源、用途、许可及商用注意事项 |
| [`summary/`](summary/2026-07-prototype-state.md) | 功能验证、评估总结与非 refactor 的重要项目决策 |
| [`others/`](others/) | 确实不能归入以上四类的文本；使用前须说明原因 |

所有文档都必须以中文 `.md`、日文 `.ja.md`、英文 `.en.md` 三语成组维护。完整规则见 [repository-doc-boundaries.instructions.md](../.github/instructions/repository-doc-boundaries.instructions.md)。
