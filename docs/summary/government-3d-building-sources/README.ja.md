# 政府建物データソースの評価

[English](README.md) | [日本語](README.ja.md) | [中文](README.zh.md)

自建 basemap と局所 3D 作業モードのための、二つの政府建物データソースの評価。
いずれもまだ未接続であり、本書は各ソースが提供する内容・ライセンス・カバレッジ・
各モードでの使い方を、確定した事実として記録する。これにより
`osm-basemap-tiles` と `local-3d-work-mode` のリファクタが確かな根拠の上で進められる。

## なぜこの二つか

OpenStreetMap の建物フットプリントは、密集都市では豊富(安定した `osm_id`、
建物種別、時に高さ)だが、日本の郊外・農村では疎である。OSM のみで描くと、地面に
建物があっても OSM が薄い場所は空に見える。以下の二つの政府ソースがその欠落を埋め、
それぞれ異なる形で実際の高さも補う。

## 基盤地図情報(国土地理院)

- **内容:** 国土地理院の国家測量基盤データ。基本項目に建築物（建築物の外周線）を
  2D ポリゴンとして含む。
- **カバレッジ:** 都市計画区域および有人地域で網羅的で、郊外・農村では OSM より
  はるかに完全。極端な山間部でのみ薄い。全国規模。
- **形式 / 入手:** JPGIS GML（Shapefile へエクスポート可）。基盤地図情報ダウンロード
  サービス（ユーザ登録要）。
- **属性:** 建物外周線。**高さは無し**（2D）、OSM 的な安定コミュニティ ID も無し
  （独自の FGD 識別子を持つ）。
- **ライセンス — 決定的な確認結果:** 基盤地図情報は基本測量成果だが、国土地理院の
  承認申請 Q&A が**明示的にこれを除外**している（地理院タイル等とともに）。ダウンロード
  して派生ベクトルタイルに加工し配布すること（商用・オフラインを含む）に**測量法の
  承認申請は不要**で、出典表示と加工表示のみでよい。プロジェクトが既に配信している
  地理院タイルと同じ扱い。出典は末尾に列挙。

## PLATEAU 建築物モデル(国土交通省 Project PLATEAU)

- **内容:** 政府の 3D 都市モデル。建築物モデルは LOD1（実測高 `bldg:measuredHeight`
  を持つ箱型）と LOD2（屋根形状）、稀に LOD4（BIM）。
- **カバレッジ:** **市町村単位**で公開。東京 23 区は全区モデル化済み。2021 年の
  56 都市から 2025 年度末に約 300 都市へ拡大。**均一ではなく**、未整備の市町村もある。
- **形式 / 入手:** CityGML（権威的な源）、3D Tiles（Cesium が描画する形式）、
  FlatGeobuf/GeoJSON 派生。G空間情報センター（CKAN）経由——UC24 地下シーンで既に
  使っているのと同じ経路。
- **属性:** `measuredHeight`（実高）、`usage`、`storeysAboveGround`、記載があれば
  `yearOfConstruction`（疎）。**建物重量は無し**（面積×高さ×階数×密度係数などで
  導出するしかなく、それは推定であって源データではない）、**施工者/開発者も無し**
  （オープンデータではない）。
- **ライセンス:** PLATEAU サイトポリシー §3（`license_id: plateau`）——出典表示、
  商用可——統合済み UC24 で既に受け入れている条件と同じ。

## 二つの表示モードでの使い方

- **地図モード(2D / 2.5D)、`osm-basemap-tiles` より。** **オフライン合成**で作る
  一つのタイルソース:OSM を主（識別子・属性）、基盤地図情報で OSM の欠落を補い
  （全国的な完全性）、モデル化済み市町村では PLATEAU の `measuredHeight` を高さ属性
  として結合。「OSM が無い所だけ政府データ」という判断を含む合成は、空間結合が
  許容できる**構築時に一度**行い、建物ごとに `footprint_source` と `height_source`
  を持つ一貫した単層を生む。これで空マップ問題（政府補完）、二重描画問題（単層化）を
  解決し、オフライン性を保ち、実測か推定かに正直であり続ける。2.5D 押し出しは
  焼き込んだ高さを起伏地表の上で読む。
- **局所 3D 作業モード、`local-3d-work-mode` より。** 領域を矩形選択し、高精細な
  PLATEAU 3D モデルをメッシュ単位でオンデマンド取得し、地上・地下を SL 重畳と AL
  シミュレーション結果とともに表示。PLATEAU のフットプリントと ID は OSM と異なる
  ため、このモードは合成タイルではなく PLATEAU を直接使い、未整備市町村では合成タイル
  の建物を押し出してフォールバックする。利用テレメトリで、関東全体を先行キャッシュ
  するのではなく、頻用領域だけをローカル化する判断を導く。

## 結論

両ソースとも、自建・オフライン対応・商用配布可能な製品にとってライセンス上クリアで、
出典表示条件で利用できる。商用ローンチ前の最終的な法務確認は、ライセンス依存の決定
として妥当である——結論に疑いがあるからではなく、決定がそれに値するから。合成タイルの
混合ライセンス（OSM ODbL + 地理院条項）は両者の表示を要し、合成**データベース**自体を
（単にタイルを提供するのでなく）公開する場合は ODbL のシェアアライクが適用される。
